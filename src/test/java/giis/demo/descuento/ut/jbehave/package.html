Tests unitarios del ejemplo de descuentos de clientes al estilo BDD (automatizados con JBehave). Hay tres componentes basicos:
  • la descripción de los escenarios de prueba (historia) en Gherkin (*-story)
  • el mapeo de los pasos escritos en Gherking a Java (*Steps.java)
  • la configuracion que indica la localización de los escenarios y del mapping a java (*Stories.java)

En maven se debe configurar el plugin jbehave-maven-plugin para soportar el uso de jbehave (ver más detalles en el pom.xml)

Para ejecutar desde Eclipse basta ejecutar como junit el archivo con la configuracion, pero no facilita la ejecución y visualizacion de los resultados en el panel de JUnit por lo que se ha instalado el plugin jbehave-junit-runner (https://github.com/valfirst/jbehave-junit-runner).
